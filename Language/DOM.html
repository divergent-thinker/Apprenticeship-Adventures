What is the Document Object Model?

The Document

The Document simply refers to the webpage.

An Object is just a thing that makes to sense to treat as an individual piece. Sometimes they contain other objects.

The Model part of it just means taking something complex and making it abstract, making it simple. In this case, taking the HTML document and creating a tree structure. Each element in this tree structure is descibed as a node. The nodes also have relationships, described as parents and siblings. So the model is an agreed upon set of terms that describe exactly how the pieces(nodes) of a webpage (the document) interact.

The DOM is the API that gives CSS and Javascript a way to modify an existing document. This allows you to write Javascript for any website. For example, you can say you want to get the text of the title, or the second paragraph, or get all the list item elements in the last unordered list, find an image with the ID of logo and change it, and so on.

The DOM is the way to reach into the page from our script and the way our page can reach into our script.

Accessing DOM elements

To grab an element using it's ID, we use the following line of code, passing in the name of the ID attribute we want: document.getElementById=("abc"). We create a variable, and set it to var myElement = document.getElementById("abc"). This variable then provides a handle to that place in the DOM.

If the element we're after doesn't have an ID, we can also use document.getELementsByTagName("li"); This is going to go through our document and get any element nodes that are li elements. We can store these in a variable again i.e. var myListItems = document.getElementsByTagName ("li");

How does it store these in one variable? It makes an array. So it makes an array called myListItems, it finds all the list items in the entire document, and then in each position of the array, we'll have a handle to that particular place in the DOM e.g. myListItem(2).

If you want to get elements from a particular part of the document, you can use that instead of document in the code. For example. You can use: var myListItems = document.getElementsByTagName("li"); to get all the list items. What if you only want the list items from an

Changing DOM attributes

For example, how do we change the href element value in click here to a different value?

There are two methods in Javascript that we can use to directly work with attributes. These are:

  1. getAttribute
  2. setAttribute

getAttribtue

for this we need to say what attribute, passed in as a string in "" e.g. class, title, src: myElement.getAttribute("src");

setAttribute

for this we need a bit more information, this time we need to name the attribute we're interested in but also provide a new value. e.g.
myElement.getAttribute("align", "left");
If the attribute doesn't currently exist it will be created.

Access the content for specific tags

First of all get the ID of the tag whose content you want to change, then store this in a variable e.g.
mainTitle = document.getElementById("mainTitle");

To print out the HTML of that tag we can then do: console.log(mainTitle.innerHTML);

Create DOM elements

Creating content requires two steps. 1. creating the DOM elements and 2. add that newly created element to the right place in the document.

  1. Create element using document.createELement("li");
  2. Next we need to call the appendChild method of the element.

Next we probably want to add some text to this element. To do this we use the createTextNode method to create text in this node. Once again we use the appendChild method except for this time we're adding it to the element we just created.

This is the most exact method for attaching text. An alternative is to use insertBefore. appendChild will always add that new list item right at the end.

If we want it as the second rather than the last list item for example we would need to do the same as before, i.e. create the new element first. e.g. var myNewElement = document.createElement("li"); var secondElement = myElement.getElementsByTagName ("li")[1]; myElement.insertBefore(myNewElement, SecondItem);

Here we pass in two parameters - the new element that we've made, and the placed we want to insert it before.

In summary, most of the time you'll use appendChild but insetBefore can also come in useful if this is not specific enough.
