Always use var when you declare a variable.

Script tags

Put the JavaScript script tag right before the closing body tag that just links to a separate JavaScript file. Two things to remember:

  1. Keep the number of script tags to a miniumum. Every JavaScript file you link to is a separate request the browser needs to make to the web server.The browers will make a call to the serve for the first JS file, then once that has finished downloading it will call the next one. It downloads sequentially rather than simultaneously. Therefore, more files you have the more requests it needs to make and the slower your page will be to load. It's therefore best to combine JavaScript files where possible rather than having multiple small files. You should have as few JavaScript files as possible.
  2. The order of the script tags matters. If you have a file of JavaScript that calls a function in a second JavaScript file, you need to ensure that file is loaded before the code that tries to call it. This is very common with libraries. e.g. It's best to load the jQuery library first, before any JavaScript that tries to use jQuery. The same holds for other dependencies.

Minifying your code

Minificaiton compacts code. e.g. removes comments, renames variable names to shorter names, puts code on one line.

Minification improves load time by reducing the file size

Tools for minification include JSMin, YUI Compressor and Google Closure Compiler. Many of these are command line tools, written in C or in Java. You can download them and run them from the command line. There are a couple that can be accessed over the web, like Google'sClosure Compiler. You can use the 'Advanced' setting to make it much smaller. The Google tool also does some optimisation. Once you've minified your code you can copy the code.

Minification makes the script much harder to read so you onlly do this once you're happy with teh script at the end. YOu should also keep a copy of your non-minified code. You can unminifiy to an extent, e.g. get whitespace back, but you cannot get old variable names back or the comments.

It's very common to find 3rd party Javascript libraries available to downlaod in both minified version and also non-minified versions (good if you want to read the code).

Javascript code checkers

You can run your JS through another programme to check it's quality. THe best code quality checker is JSLint, created by Douglas Crockford (JS architect at Yahoo). You paste your code in and click the 'JSLint' button to check it.

JS Lint is very fussy and will point out ALL of the imperfections. There are options at the bottom to tell JS Lint to tolerate certain things. e.g. 'Assume a browser' tells it to assume you're in a browser (since JS might be run elsewhere).
