Application Programming Interface is usually abbreviated to API.

It's something that allows one piece fo software to talk to another. An API is the messanger that takes requests, tells a system what you want to do and then returns the results back to you.

REST APIs

A REST API stands for Representational State Transfer. Usually this works the same way as a website does. You make a call from a client to a server to get data back over the HTTP protocal.

Google maps API

The Google maps API can return data in a number of formats. .e.g. can input the url maps.googleapis.com/maps/api/geocode/json

In this case the server that we're calling is maps.googleapis.com. The particular resource (we've drilled down) is maps resource, then the api resource, then the geocode resource and the JSON resource. Then we could add parameters e.g. we could put maps.googleapis.com/maps/api/geocode/json?address=chicago&sensor=false. This will give us a JSON response with the information we need. We could then take that data and pass it into another API e.g. instragram to grab a photo from Chicago.

There are thousands of APIs that you can tap into, take their data and use it pass it to another API. Many websitse have some kind of API that you can use in order to consume their data. Check out: ProgrammableWeb

Writing data to APIs: HTTP Request Methods

In order to write data to APIs you need to know a bit more about HTTP request methods. Two of the mains ones are GET and POST. A GET request is what you use to request data as discussed above using these URL parameters in order to get data back from the API. A POST request is for writing data to the API. The best practice is to put the data in the actual body of the request. To do this you can install an extension called 'Postman' client. See Postman demo.
