Handlebars

Handlebars is a templating engine which allows you to build semantic templates. It is based on Mustache template language, but improves on it in a number of key areas.

Installation

See here for Handlebars installation instructions

To install via terminal user npm install --save handlebars

Once you have installed handlebars, you will need to require it in your JS file e.g. var handlebars = require('handlebars');

Basics Expressions

Handebars templates look like normal HTML documents, with embedded handbars expressions. The handlebars expression is a {{some content followed by a }}. When an expression is reached, handlebars will look for an item in the current context that matches it. If the matching item is a value, that will be the output. IF the matching item is a function, it will be called.

A dot (.) operator can be used in expressions to output nested values e.g. {user.firstname} would output the firstName property on the user value in the current context. Note it should be a double not single moustache, I just had to write it like this to get it to display.

See this blog for more detail.

Blocks

Blocks can be used to template, er, blocks in a template. For example, you might have a list of names that you wish to display, which are held in an array.

To open and close a block use a # and a close tag e.g.:

    {#people}}
  • {name}}
    • {/people}}
This would output the following array for example: var data = {people: [
{name: "Alan"},
{name: "Allison"},
{name: "Ryan"}
], group: "Bloggers" };
